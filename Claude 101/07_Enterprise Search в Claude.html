Enterprise Search в Claude: поиск знаний по всей организации

Цели обучения

По завершении этого урока вы сможете:

  • объяснять, что такое Enterprise Search и какие типы вопросов он помогает решать;
  • понимать, как проходит процесс настройки Enterprise Search для администраторов и пользователей;
  • осознавать, как устроены безопасность и контроль доступа к данным организации.

Что такое Enterprise Search

Enterprise Search (Корпоративный поиск) — это специализированный режим работы Claude, который добавляет в боковую панель отдельную опцию
«Спросить {Название вашей организации}».

Он предназначен для поиска, объединения и обобщения знаний, распределённых по внутренним инструментам и источникам данных компании. По сути, Enterprise Search можно рассматривать как заранее подготовленный проект для всей организации, где корпоративная база знаний уже подключена и готова к использованию.

В отличие от обычных чатов с включёнными коннекторами, Enterprise Search:

  • ориентирован именно на поиск и синтез информации;
  • работает по специально настроенным инструкциям;
  • оптимизирован под организационный контекст.

Какие вопросы можно задавать

Enterprise Search особенно полезен, когда требуется:

  • собрать информацию из нескольких источников;
  • обобщить знания, разбросанные по разным отделам и системам;
  • быстро сориентироваться в текущем состоянии дел.

Вхождение в курс дел

  • «Что произошло вчера, пока меня не было?»
  • «Суммируй ключевые обновления в бизнесе за последнюю неделю»
  • «Какие блокеры сейчас есть у проекта Platform?»

Политики и процессы

  • «Какова политика удалённой работы в нашей компании?»
  • «Как подать отчёт о расходах?»
  • «Каков процесс запроса отпуска?»

Исследования и аналитика

  • «Почему клиенты выбирают конкурентов?»
  • «Суммируй обсуждения дорожной карты продукта на Q4»
  • «Найди информацию о процессе онбординга клиентов»

Онбординг сотрудников

  • «Как работает система аутентификации?»
  • «К кому обратиться по вопросам биллинга?»
  • «Какие инструменты использует инженерная команда для деплоя?»

Отслеживание проектов и эффективности

  • «Найди обсуждения и документы по маркетинговой кампании»
  • «Какие ключевые решения были приняты на недавних встречах руководства?»
  • «Суммируй вклад команд в инициативу Infrastructure»

Как Claude отвечает на запросы

При использовании Enterprise Search Claude:

  • ищет данные во всех подключённых корпоративных источниках
    (например, SharePoint, Slack, Gmail, Google Drive);
  • обобщает найденную информацию в единый ответ;
  • всегда указывает источники, чтобы вы могли проверить контекст и первоисточники.

Это делает ответы не только удобными, но и проверяемыми.

Настройка Enterprise Search

Настройка проходит в два этапа:

  1. начальная конфигурация администратором;
  2. подключение сервисов отдельными пользователями.

Настройка для администраторов (Owners)

Для Team и Enterprise‑организаций Enterprise Search включён по умолчанию, но администратор должен завершить первичную настройку.

Шаги:

  1. Нажмите «Ask Your Org» в боковой панели.
  2. Выберите «Set up for your org».
  3. Подключите инструменты организации:
    • документы (Google Drive или SharePoint);
    • коммуникации (Slack или Microsoft Teams);
    • электронную почту (рекомендуется, но необязательно).
  4. При необходимости добавьте дополнительные сервисы через Add more.
  5. Задайте имя проекта — именно оно появится у сотрудников в виде «Ask [Имя]».
  6. Добавьте описание и завершите настройку.

После этого Enterprise Search становится доступным для всех членов организации.

Настройка для пользователей

После завершения настройки администратором:

  1. В боковой панели появится проект Ask {Org Name}, отмеченный звездой.
  2. Откройте проект.
  3. Пройдите пошаговый онбординг:
    • подключите рекомендуемые сервисы;
    • выполните авторизацию в каждом из них.
  4. Начните задавать Claude вопросы о знаниях вашей организации.

Чем больше сервисов вы подключите, тем полнее и точнее будут ответы. Подключения можно добавлять позже через раздел инструкций проекта.

Безопасность и доступ к данным

Безопасно ли это?

Короткий ответ — да.

Enterprise Search строго соблюдает принципы безопасности:

  • Claude видит только те данные, к которым у вас уже есть доступ в исходных системах;
  • он не получает дополнительных привилегий;
  • данные не индексируются и не сохраняются отдельно;
  • диалоги пользователей остаются приватными.

Например, подключение вашей рабочей почты не даёт Claude доступа к почте руководства — только к вашему почтовому ящику.

Доступ к сервисам можно в любой момент отозвать через настройки Claude или самого сервиса.

Основные выводы урока

  • Enterprise Search превращает корпоративные данные в доступную и удобную систему ответов.
  • Он особенно полезен для онбординга, аналитики и навигации по внутренним процессам.
  • По сути, это «единый вход» ко всем знаниям организации.
  • Безопасность и контроль доступа полностью сохраняются.
  • Ответы всегда сопровождаются указанием источников.
